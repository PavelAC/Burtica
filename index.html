Descriere Generală

🎯 Scopul Proiectului

Dezvoltarea unei baze de date normalizate pentru un sistem de management al serviciilor de livrare, care să permită gestionarea eficientă a comenzilor, produselor, clienților și curierilor.

🏗️ Arhitectura Sistemului

Sistemul utilizează o structură relațională normalizată care respectă principiile ACID și asigură integritatea datelor prin chei primare și străine.

⚡ Funcționalități Principale

  • Gestionarea catalogu de produse cu categorii
  • Procesarea comenzilor cu tracking în timp real
  • Management curierii și vehicule
  • Sistema de review-uri și rating
  • Gestionarea clienților și istoricul comenzilor
  • Raportare și analiză performanțe

🎯 Logica Structurii Finale

Principii respectate:

  • Normalizare 3NF - eliminarea redundanței
  • Integritate referențială - chei străine
  • Atomicitate - fiecare câmp indivisibil
  • Consistență - constrainte de validare

Relații implementate:

  • One-to-Many: categorie → produs
  • Many-to-Many: produs ↔ comanda
  • One-to-Many: client → comanda
  • One-to-One: comanda → tracking
Scenarii de Utilizare
Specificații Funcționale cu Comenzi MySQL

🛒 SC-01: Plasarea unei comenzi

Clientul selectează produse, specifică cantitățile și confirmă comanda.

-- Inserarea unei noi comenzi
INSERT INTO comanda (total_comanda, curier_id_curier, client_id_client) 
VALUES (125.50, 1, 3);

-- Adăugarea produselor la comandă
INSERT INTO produs_has_comanda (produs_id_produs, comanda_id_comanda, cantitate) 
VALUES (5, LAST_INSERT_ID(), 2), (8, LAST_INSERT_ID(), 1);

-- Actualizarea stocului
UPDATE produs SET stoc = stoc - 2 WHERE id_produs = 5;
UPDATE produs SET stoc = stoc - 1 WHERE id_produs = 8;

-- Crearea tracking-ului
INSERT INTO tracking (status, update_time, id_comanda) 
VALUES ('Comanda plasată', NOW(), LAST_INSERT_ID());

🚚 SC-02: Procesarea livrării

Curierul actualizează statusul comenzii în timpul procesului de livrare.

-- Actualizarea statusului la "În curs de livrare"
UPDATE tracking 
SET status = 'În curs de livrare', update_time = NOW() 
WHERE id_comanda = 15;

-- Confirmarea livrării
UPDATE tracking 
SET status = 'Livrat', update_time = NOW() 
WHERE id_comanda = 15;

-- Verificarea comenzilor curierului
SELECT c.id_comanda, cl.nume as client, c.total_comanda, t.status
FROM comanda c
JOIN client cl ON c.client_id_client = cl.id_client
JOIN tracking t ON c.id_comanda = t.id_comanda
WHERE c.curier_id_curier = 1;

📊 SC-03: Gestionarea stocurilor

Monitorizarea și actualizarea stocurilor de produse.

-- Verificarea produselor cu stoc scăzut
SELECT p.nume_produs, p.stoc, cat.nume as categorie
FROM produs p
JOIN categorie cat ON p.id_categorie = cat.id_categorie
WHERE p.stoc < 10
ORDER BY p.stoc ASC;

-- Reaprovizionarea stocului
UPDATE produs 
SET stoc = stoc + 50 
WHERE id_produs IN (2, 5, 8);

-- Raport stoc pe categorii
SELECT cat.nume, COUNT(*) as nr_produse, SUM(p.stoc) as total_stoc
FROM categorie cat
JOIN produs p ON cat.id_categorie = p.id_categorie
GROUP BY cat.nume;

Analize și Raportare

📈 SC-04: Raportare vânzări

Generarea rapoartelor de vânzări pe categorii și perioade.

-- Vânzări pe categorii în ultima lună
SELECT cat.nume as categorie, 
       COUNT(DISTINCT c.id_comanda) as nr_comenzi,
       SUM(phc.cantitate) as total_produse_vandute,
       ROUND(AVG(c.total_comanda), 2) as medie_comanda
FROM categorie cat
JOIN produs p ON cat.id_categorie = p.id_categorie
JOIN produs_has_comanda phc ON p.id_produs = phc.produs_id_produs
JOIN comanda c ON phc.comanda_id_comanda = c.id_comanda
WHERE c.data >= DATE_SUB(NOW(), INTERVAL 1 MONTH)
GROUP BY cat.nume
ORDER BY total_produse_vandute DESC;

-- Top 5 produse cele mai vândute
SELECT p.nume_produs, cat.nume as categorie, 
       SUM(phc.cantitate) as total_vandut,
       ROUND(SUM(phc.cantitate * p.pret), 2) as venituri_totale
FROM produs p
JOIN categorie cat ON p.id_categorie = cat.id_categorie
JOIN produs_has_comanda phc ON p.id_produs = phc.produs_id_produs
GROUP BY p.id_produs
ORDER BY total_vandut DESC
LIMIT 5;

⭐ SC-05: Evaluarea performanței

Analiza performanței curierilor și satisfacției clienților.

-- Performanța curierilor
SELECT cur.nume as curier, cur.vehicul,
       COUNT(c.id_comanda) as nr_comenzi_livrate,
       ROUND(AVG(r.rating), 2) as rating_mediu,
       COUNT(r.id_review) as nr_review_uri
FROM curier cur
JOIN comanda c ON cur.id_curier = c.curier_id_curier
LEFT JOIN review r ON c.id_comanda = r.id_produs
GROUP BY cur.id_curier
ORDER BY rating_mediu DESC;

-- Analiza timpilor de livrare
SELECT 
    CASE 
        WHEN TIMESTAMPDIFF(HOUR, c.data, t.update_time) <= 24 THEN '≤ 24h'
        WHEN TIMESTAMPDIFF(HOUR, c.data, t.update_time) <= 48 THEN '24-48h'
        ELSE '> 48h'
    END as timp_livrare,
    COUNT(*) as nr_comenzi
FROM comanda c
JOIN tracking t ON c.id_comanda = t.id_comanda
WHERE t.status = 'Livrat'
GROUP BY timp_livrare;

👥 SC-06: Managementul clienților

Analiza comportamentului și istoricului clienților.

-- Profilul clienților fideli
SELECT cl.nume, cl.email,
       COUNT(c.id_comanda) as nr_comenzi,
       ROUND(SUM(c.total_comanda), 2) as total_cheltuit,
       ROUND(AVG(c.total_comanda), 2) as medie_comanda,
       MAX(c.data) as ultima_comanda
FROM client cl
JOIN comanda c ON cl.id_client = c.client_id_client
GROUP BY cl.id_client
HAVING nr_comenzi >= 3
ORDER BY total_cheltuit DESC;

-- Preferințele clienților pe categorii
SELECT cl.nume as client, cat.nume as categorie_preferata,
       COUNT(*) as nr_comenzi_categorie
FROM client cl
JOIN comanda c ON cl.id_client = c.client_id_client
JOIN produs_has_comanda phc ON c.id_comanda = phc.comanda_id_comanda
JOIN produs p ON phc.produs_id_produs = p.id_produs
JOIN categorie cat ON p.id_categorie = cat.id_categorie
GROUP BY cl.id_client, cat.id_categorie
ORDER BY cl.nume, nr_comenzi_categorie DESC;
